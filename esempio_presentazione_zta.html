Perché Zero Trust?

Il confine di rete tradizionale è dissolto. Un approccio data-centrico è mandatorio.

Il Caso di Studio

Protezione di un database MongoDB ospedaliero contenente cartelle cliniche. Accesso richiesto da rete interna e smart working.

  • Dati sanitari critici (Compliance GDPR, NIS2)
  • Connessioni da reti eterogenee
  • Rischio costante di insider threat

Il Paradigma Zero Trust

"Never Trust, Always Verify"

  • Nessun attore è implicitamente fidato
  • Autorizzazione per-request continua
  • Analisi contestuale dinamica e telemetria

Le 6 Dimensioni Zero Trust

Ogni richiesta è un vettore analizzato dal PDP.

👤 User (u)

Estrazione dell'identità tramite Common Name del cert mTLS.

🖥️ Device (d)

Hard-token verification tramite OID TPM 1.3.6...1.

💻 Software (s)

Fingerprint JA3 tramite hashing del client hello (TLS).

🌐 Network (n)

Provenienza geografica o di rete (IP translation Envoy).

⚡ Action (a)

Comando API o verbo BSON MongoDB richiesto.

📁 Resource (r)

Target finale dell'azione (Collection o Path HTTP).

OPA: Motore Decisionale Rego

Analisi stateless e crittografica sui certificati inoltrati da Envoy.

Logica Declarativa

  • Costrutto default allow = false.
  • Parsing X.509 ASN.1 tramite Rego.
  • Ispezione di estensioni custom (TPM OID).
  • Interrogazione WebAPI (Splunk) in tempo reale via http.send.
rules.rego 
# Validazione chip TPM hardware
is_tpm := true if {
  cert_pem := urlquery.decode(cert_raw)
  certs := crypto.x509.parse_certificates(cert_pem)
  some ext in certs[0].Extensions
  ext.Id == [1,3,6,1,4,1,9999,1]
}

NFTables: Perimetro L3/L4

Difesa perimetrale iniziale ed enforcement del routing forzato.

Policy Base

  • DNAT: Dirottamento del traffico autorizzato verso Envoy.
  • Masquerading: SNAT per le risposte interne.
  • Drop & Log: Blocco silenzioso e logging per bypass API diretti.
start.sh 
# Routing sicuro verso Envoy (PEP)
nft add rule ip nat prerouting \
  tcp dport 8443 counter dnat to $ENVOY:8443

# Prevenzione bypass verso backend ML
nft add rule ip nat prerouting \
  tcp dport 8000 drop
